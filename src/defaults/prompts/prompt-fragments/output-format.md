## Output Formatting Rules

<output_format>
- Be concise and direct.
- Use structured formats (lists, tables, code blocks) for complex data.
- Clearly distinguish success from failure in results.
- For errors, include: error type, message, and suggested fix.
- When returning data, use JSON format unless instructed otherwise.
- Do not include unnecessary preamble or filler text.
</output_format>

<text_formatting>
Use standard Markdown for formatting. Avoid these incompatible patterns:

- Do NOT use `__double underscores__` - use `**asterisks**` for bold instead
- Do NOT use HTML tags (`<b>`, `<i>`, etc.) - use Markdown syntax
- Do NOT use `||spoiler||` syntax - not universally supported

Keep formatting minimal and readable.
</text_formatting>
